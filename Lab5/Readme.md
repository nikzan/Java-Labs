## Структура проекта
Проект посвящен изучению шаблонов проектирования (Proxy, Decorator), коллекций и Stream API.
- `Main` — главный класс с консольным меню для проверки всех заданий.
- `Fraction` и `CachedFraction` — реализация дробей и кэширования (Proxy).
- `Cat` и `MeowCounter` — реализация котов и подсчета мяуканий (Decorator).
- `ListUtils`, `MapUtils`, `SetUtils`, `QueueUtils` — утилитарные классы для работы с коллекциями.
- `StreamTasks` — класс с задачами на Stream API (ломаная, группировка людей).
- `Point`, `Polyline` — вспомогательные классы для геометрии.

## Как работает основной код в Main
Класс `Main` запускает цикл с меню, где пользователь может выбрать номер задания (1-7) или выйти (0).
Для заданий, требующих ввода данных (например, удаление элемента из списка), программа запрашивает ввод с консоли.
Для заданий с файлами (`students.txt`, `text.txt`, `people.txt`) программа автоматически читает данные из корневой папки проекта.

## Задача 1 — Дроби и Кэширование
Реализация: пакет `ru.nikzan.fractions`.
### Назначение
Создание обертки над дробью, которая кэширует ее вещественное значение (`doubleValue`), чтобы не пересчитывать его каждый раз.
### Классы
- `Fraction` — обычная дробь (числитель, знаменатель).
- `FractionInterface` — общий интерфейс.
- `CachedFraction` — прокси-класс. Хранит вычисленное значение. Сбрасывает кэш при изменении числителя или знаменателя.

**Примеры тестирования:**
```
Fraction f = new Fraction(3, 4);
CachedFraction cf = new CachedFraction(f);
cf.doubleValue() -> 0.75 (вычисление)
cf.doubleValue() -> 0.75 (из кэша)
cf.setNumerator(1); cf.setDenominator(2);
cf.doubleValue() -> 0.5 (пересчёт после изменения)
```

## Задача 2 — Коты и Мяуканье
Реализация: пакет `ru.nikzan.cats`.
### Назначение
Подсчет количества мяуканий кота без изменения кода самого кота.
### Классы
- `Meowable` — интерфейс для мяукающих объектов.
- `Cat` — обычный кот, который выводит "Meow!" в консоль.
- `MeowCounter` — декоратор. При вызове `meow()` увеличивает счетчик и вызывает метод у оригинального кота.

**Примеры тестирования:**
```
Cat cat = new Cat("Барсик");
MeowCounter counter = new MeowCounter(cat);
counter.meow(); // Вывод: Meow!
counter.meow(); // Вывод: Meow!
counter.meow(); // Вывод: Meow!
counter.getCount() -> 3
counter.meow(); counter.meow();
counter.getCount() -> 5
```

## Задача 3 — Список
Реализация: `ru.nikzan.collections.ListUtils`.
### Описание
Метод принимает список целых чисел и элемент, который нужно удалить.
Удаляет все вхождения этого элемента из списка.
Используется итератор для безопасного удаления во время прохода по списку.

**Примеры тестирования:**
```
List<Integer> list = [1, 2, 3, 2, 4, 2, 5];
ListUtils.removeAll(list, 2);
list -> [1, 3, 4, 5]

List<Integer> empty = [];
ListUtils.removeAll(empty, 1);
empty -> []

List<Integer> noMatch = [1, 3, 5];
ListUtils.removeAll(noMatch, 2);
noMatch -> [1, 3, 5]
```

## Задача 4 — Map (Логины)
Реализация: `ru.nikzan.collections.MapUtils`.
### Описание
Генерация уникальных логинов для студентов из файла `students.txt`.
Используется `Map` для подсчета количества повторений каждой фамилии.
- Первый раз: `Ivanov`
- Второй раз: `Ivanov2`
- Третий раз: `Ivanov3`

**Примеры тестирования:**
```
Вход (students.txt):
Иванов
Петров
Иванов
Сидоров
Иванов
Петров

Выход:
Ivanov
Petrov
Ivanov2
Sidorov
Ivanov3
Petrov2
```

## Задача 5 — Set (Звонкие согласные)
Реализация: `ru.nikzan.collections.SetUtils`.
### Описание
Поиск всех звонких согласных букв в тексте из файла `text.txt`.
Используется `TreeSet` для автоматической сортировки букв в алфавитном порядке и исключения дубликатов.

**Примеры тестирования:**
```
Вход (text.txt): "Белый бор блестит в лесу"
Звонкие согласные: б, в, д, ж, з, л, м, н, р
Выход: [б, в, л, р] (отсортированные уникальные)

Вход: "Тихий шепот"
Выход: [] (нет звонких согласных)
```

## Задача 6 — Очередь
Реализация: `ru.nikzan.collections.QueueUtils`.
### Описание
Метод переворачивает очередь (элементы идут в обратном порядке).
Для реализации используется вспомогательный стек (`Stack`), так как он работает по принципу LIFO (последний пришел — первый ушел).

**Примеры тестирования:**
```
Queue<Integer> queue = [1, 2, 3, 4, 5];
QueueUtils.reverse(queue);
queue -> [5, 4, 3, 2, 1]

Queue<String> names = ["Иван", "Петр", "Сидор"];
QueueUtils.reverse(names);
names -> ["Сидор", "Петр", "Иван"]
```

## Задача 7 — Stream API
Реализация: пакет `ru.nikzan.streams`.
### 7.1 Ломаная
Обработка списка точек:
1. Удаление дубликатов (`distinct`).
2. Сортировка по координате X (`sorted`).
3. Преобразование Y в положительное число (`map`).
4. Сборка в объект `Polyline`.

**Примеры тестирования:**
```
Вход: [Point(3, -5), Point(1, 2), Point(3, -5), Point(2, -3)]
После distinct: [Point(3, -5), Point(1, 2), Point(2, -3)]
После sorted: [Point(1, 2), Point(2, -3), Point(3, -5)]
После map: [Point(1, 2), Point(2, 3), Point(3, 5)]
Результат: Polyline{points=[Point(1, 2), Point(2, 3), Point(3, 5)]}
```

### 7.2 Группировка людей
Чтение файла `people.txt` с форматом `Имя:Номер`.
1. Имена приводятся к виду "С большой буквы" (Capitalize).
2. Люди группируются по номеру.
3. Результат возвращается в виде `Map<Integer, List<String>>`.

**Примеры тестирования:**
```
Вход (people.txt):
иван:1
петр:2
сидор:1
мария:3
анна:2

Результат:
1 -> [Иван, Сидор]
2 -> [Петр, Анна]
3 -> [Мария]
```
